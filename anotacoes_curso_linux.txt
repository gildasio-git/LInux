 ++ DE ONDE VEIO O LINUX ++

 O Inicio em 1965 nos laboratórios da BELL LABS, que pertencia a AT&T, para quem não conhece ou não sabe, os transistores
 foram criados lá nessa empresa, começou desenvoler um projeto de um sistema operacional de tempo compartilhado que era 
 MULTICS, era um projeto de empresa junto com o governo Americano. em 1969 A BELL saiu do projeto e começou a desenvolver
 seu próprio sistema operacional chamado UNIX (UNIPLEXED INFORMATION AND COMPUTING SYSTEM), dois desenvolvedores e em particular
 DENNIS RITCHIE que faleceu a pouco tempo, das mãos desse homem sairam a linguagem C e UNICS, foi um dos primeiros compiladores da 
 linguagem B, o outro desenvolvedor KEN TOMPSON, o sitema estava sendo feito em ASSEMBLER, linguágem de máquina, porém a linguagem
 não é portável ou seja, na época não rodava em comptuadores de diferente arquitetura, teria que reescrever o código. entãnto para isso
 eles portaram o sistema para uma linguagem chamada BCPL, dessa linguagem que posteriormente virou a linguagem B, após tantas modificações
 na linguagem rebatizaram para linguagem C. (evolução da linguagem A).

 Em 1973 o UNIX se tornou forma, mdou a nomenclaruta que era UNICS se tornando UNIX com "X". para esclarecimento em 1970a BEL LABS que pertencia 
 a AT&T estava em um processo antitrush, processo relacionado a monópolio no mercado de telefonia, possuindo 60% do mercado americano
 o que impossibiliou a venda do UNIX, caso começacem vender esse sistema a investigação do dpto de comércio os acusariam de exercer
 também monopólio em sistemas de computadores.

 Então A BELL LABS decidiou passar o sistema para quem quisar porém sem suporte, caso  voce quisesse uma cópia do sistema operacional 
 teria que enviar uma carta para a AT&T, uma fita de 9 mm então eles copiavam o sistema nessa fita. O que aconteceu? muitas empresarias
 que pegaram o sistema, (código fonge) foram adaptando as suas necessidades, dai surgiram vários sabores do sistema operacional. dessas
 um dos sistemas mais famoso da humanidade foi o FREEEBSD 4.4. uma versão que foi levada para Universidade BERCLEY na Califórnia, então
 quando ouvidos falar de FREEBSD, NETBSD e o próprio núcleo DARVIN do MACOS 10, uma curiosidade é que foi desenvolvida a arquitetura TCP/IP
 que é a base da internet que conhecemos, década de 70, como várias outras estruturas.com essa evolução comeram a surgir várias vesões
 do sistemas, a própria IBM possuia uma versão chamada AIX. exemplos como SOLARIS da SUN, IRIX DA SILICON GRAPHICS, AIX DA IBM, MACOS 10
 XENIX, USIX.


 FIM DOS ANOS 70, AT&T JA HÁVIA SIDO DIVIDIDA, COM A CRIAÇÃO DAS BELLs(EMRPESAS MENORES), ENTÃO DECIDIRAM COBRAR DE QUEM ESTAVA USANDO 
 O SISTEMA, UMA BRIGA QUE SE ARRASTOU DO FINAL DO ANOS 70 A MEIO DOS ANOS 90 NA JUSTIÇA AMERICANA. ENTÃO PARA DEFNIIR SE UM SISTEMA FOSSE
 PADRÃO UNIX, DEVEIRA OBEDECER UM CONJUNTO DE REGRAS, E QUEM DETERMINA ESSAS REGRAS? UM COMITÊ CHAMADO POSIX ( PORTABLE OPERATION SYSTEM 
 INTERFACE BASE ON UNIX)  LIGADO A IEEE (INSTITUTO DE ENGENHEIROS ELÉTRICOS ELETRÔNICOS), PARA QUEM NÃO SABE O WINDOWS OBEDECE VÁRIAS DESSAS
 REGRAS, QUE DETERMINA SE UM SISTEMA É UNIX.

 NO FINAL DOS ANOS 70 COMEÇOU A BRIGA NA JUSTIÇA, SURGIU EMPRESAS QUE QUERIA TRABALHAR COM ISSO, ENTÃO SURGIU UM CIDADÃO, (HACKER VELHO)
 BARBUDO, CABELUDO, (RICARHD STALLMAN) INCLUSIVE UM AMIGO QUE HOSPEDOU O STALLMAN NO RIO DE JANEIRO, SOMENTE APENASU UMA SEMANA PEDIU 
 UM BALDE PARA LAVAR O CABELO. OU SEJA NÃO TOMOU BANHO. INCLUSIVE DURANTE UMA PALESTRA O MESMO TIROU CRAKA DO PÉ E COMEU.

 ENTÃO NA ÉPOLCA ESSE MESMO HACKER VELHO ERA PESQUISADOR CENTRO DE INTELIGÊNCIA ARTIFICIAL DO MIT (MASSACHSETTS INSTITUTE OF TECHNOLOGY) 
 E VINHA DE UMA LONGA TRADIÇÃO HACKER. O QUE TINHA DE CONCEITO NA ÉPOCA ERA QUE AS EMPRESAS NÃO VENDIAM SOFTWARES, VENDIAM HARDWARE
 O SOFTWARE ERA UM ACESSÓRIO. (A MICROSOFT QUE COMEÇOU COM A POLITICA DE VENDER SOFTWARE), ENTÃO COMO STALLMAN VINHA DE UMA TRADIÇÃO
 NÃO DE VENDER SOFTWARE MAIS SIM COMPARTILHAR CÓDIGO, ENTÃO LOGO ELE PENSOU, "APRENDI VENDO CÓDIGO DOS OSUTROS " E AGORA  COMO SERÁ O
 FURUTO DAS NOVAS GERAÇÕES COM ESSA POLITICA DE CÓDIFO FECHADO, ENTÃO VVEM AI O CÚMULO DA HISTÓRIA DA IMPRESSORA, A XEROX DOOU UMA
 IMPRESSORA PARA O LABORATÓRIO QUE ELE TRABALHA NO MIT, E HAVIA UM PROBLEMA DE UM SENSOR QUE AS VEZES O PAPEL ENRROLAVA E COMO ERA UMA
 IMPRESSORA SÓ, TERIA QUE LEVANTAR A TODO MOMENTO PARA IR RESOLVER. ENTÃO ELE SOLICITOU A XEROS UMA CÓPRIA DO DRIVER PARA FAZER ALTERAÇÕES
 PARA CASO O PAPEL EMBOLASSE PARA QUE A IMPRESSORA OS AVISASSEM. A XEROX NÃO FORNECER O CÓDIGO FONTE, ESSE FATO ACONTECER POR VOLTA DE 1982
 ENTÃO ELE ARGUMENTOU QUE NÃO TERIA PROBLEMA ELE ASSINARIA UM CONTRATO DE QUE NÃO DIVULGARIA O QUE TINHA NO CÓDIGO, MESMO ASSIM A XEROX
 SE NEGOU A FORNECER O CÓDIGO FONTE DA IMPRESSORA. O QUE FOSSE A GOTA DE ÁGUA.

 NO ANO DE 1983 ESSE CIDADÃO PUBLICOU O MANIFESTOU GNU, (GNU NOT IS UNIX ) E DECLAROU QUE TINHA INTERESSE EM PRODUZIR UM SISTEMA OPERACIONAL
 PADRAO UNIX COMPLETAMENTE LIVRE, SO LEMBRANDO O UNIX NÃO ERA LIVRE, FOI DISTRIBUIDO E DEPOS PASSOU A COBRAR (MUDOU DE IDEIA).
 DECORRENTE A ISSO A BRIGA JUDICIAL SOBRE AS LINCEÇAS DO UNIX ESTENDEU ATÉ 1995 O DEIXANDO O UNIX PARADO AGUARDANDO O QUE IRIA ACONTECER.
 ENTÃO STALLMAN DECIDIU FAZER O SISTEMA OPERACIONAL, A SABER, UM SISTEMA OPERACIONAL E COMPOSTO POR TRÊS PARTES 
 UM NUCELO DE SISTEMA
 UM CONJUNTO DE BIBLIOTECAS PARA PODER ACESSAR O NÚCLEO
 UM CONJUNTO DE UTILITÁRIOS

 O QUE ELE DECIDIU, ELE FEZ AS BIBLIOTECAS E COMEÇOU A REGIMENTAR COM O QUE TINHA DE INTERNET, COMEÇOU A ENVOLVER OUTRAS PESSOAS NO PROJETO
 FIZERAM O CONJUNTO DE UTILITÁRIOS,  FALTANDO APENAS O NÚCLEO DO SISTEMA (HURD) A COISAS ANDARAM AE 1991 QUANDO ENTRA O LINUX TORVALDS NA HISTÓRIA.
 SO PARA CONSTAR, STALLMAN SAIU DO MIT E CRITOU A FUNÇÃO SOFTWARE LIVRE (FSF), O PROJETO GNU E UM PROJETO DA FSF.FSF.ORG.

 1991 - LINUX TORVALDS ERA UM ESTUDANTE DE COMPUTAÇÃO NA UNIVERSIDADE DE HELSINK, NERD DO TIPO CLÁSSICO. ELE GANHOU UM 80386 PROCESSADOR DA ÉPOCA
 ENTÃO COMEÇOU A VER UM TRABALHO DE UM PROFESSOR CHAMADO ANDREW TANEBAUM, QUE DESENVOLEU UM SISTEMA CHAMADO MINIX (MINI UNIX).
 ENTÃO ELE COMEÇOU A MECHER, FAZER ALTERAÇÕES, ESTUDAR ESSE MINI SISTEMA OPERACIONAL, CHEGOU UM PONTO QUE ELE ESBARROU NA LICENÇA, PORQUE ELE

 MUITO AVANÇADA.

 ENTÃO LINUX RESOLVER DESENVOLVER SEU PRÓPRIO SISTEMA OPERACIONAL, ENTÃO EM OUTUBRO DE 1991 LINUX SOLTOU A PRIMEIRA VERSÃO DO KERNEL

 O NOME "LINUX", E ELE CONCORDOU, ATÉ HOJE EXISTE O FTP DESSE PROJETO, ONDE O LINUX COMEÇOU.

 ENTÃO ELE COMEÇOU A DESENVOVER, ABRIU O PROJETO, LIBEROU NA USINET NOS GRUPOS DE DISCURSÃO E FALOU "OLHA" TO COMEÇANDO A FAZER QUEM QUISER

GNU/LINUX - DEBIAN

Porque o Debian?

 - Linux como Sistema ÚNICO. (Maquina mais modesta)
 - Linux com Dual BOOT (Usando particionamento)
 - Fazer uma Instalação usando uma Máquina Virtual.		

Debian é uma das distribuições mais antigas (1994) desenvolvida por um grupo de mais de 200 pessoas
então seguramente, é um projeto cooperativo de software livre do mundo.

DEB - Débora
IAN - Ian Murdock

Ian recentemente veio a falecer porém a distro continua no mercado, e amplamente usado
DEBIAN é uma das distribuições talvez mais usadas do mundo, suas versões são somente lançadas
após muitos testes, log e considerada pronta quando relamente esta pronta.
Não existe pressão para sair uma nova  versão a cada 6 meses ou a cada 3 meses.

Uma versão da DEBIAN leva certa de 3 ou quatro anos para sair, possui uma política de 
desenvolvimento em três níveis.
 - STABLE (Estável)
 - INSTABLE (Instável)
 - TESTING (Teste)

ESTÁVEL
Versão mais antiga, com pacotes obsoletos, mais seguramente estável, com pouquíssima
chance de ter problemas.

INSTÁVEL
São pacotes mais novos, versões mais atualizadas, porém ainda poderá ocorrer alguns 
problemas.

TESTE
Loteria, mais chances de ocorrer ocorrer  problemas.

DEBIAN não é mantida  por empresa (site: www.debian.org), é uma fundação que apoia
a SPI  que banca a DEBIAN, possui suporte a mais de 12 arquiteturas, e 15 sub arquitetuas.

X86
AMD 64 
DEC ALPHA
VMS
SPARK
MACHINTOCH CLÁSSICO
RPC 
ARM
ARM64

Debian é tão flexível que caso queira, pode usar um KERNEL BSD, HURD, FREE BSD, as 
atualizações dela são constantes, não é necessário possuir um CD para fazer UPGRADE
o que é legal, pode ir mantendo atualzizado, há e uma curiosidade: todos os nomes
das verões da DEBIAN são nomes dos personagens da animação TOY STORY.
exemplo, a versão usada nesse curso é  DEBIAN STRECH, porém ja teve o SQUEEZE, no momento
está na versão 9 será o BUSTER o cachorro, a versão teste é o CID, o  garotinho malvado que quebra os brinquedos.

PROCESSOS NO LINUX 

Estados do Processo
- EXECUTANDO
- ESPERANDO
- PRONTO
- SUSPENSO
- NÃO SUBMETIDO
- COMPLETO

Processo ( uma instância de um programa de computador que esta sendo executado)

	Esta instância tem ó código do programa que esta sendo executado, como sua 
atividade atual. tem o programa EXECUTÁVEL e traz junto um conjunto de informações
que são necessárias para a execução to processo.

	Em resumo, é o programa sendo executado e o ambiente em torno dele.
	
	Possui:
	o Código executável
	o Espaço de Endereçamento (Região de memória usada por ele)
	o Dados do S.O (os outros processo que aquele processo criou)
	o Arquivos que foram abertos por ele
	o Atributos no que trata de Permissões que ele tem
	o Questões do Estado da CPU (Questões relacionadas ao registrador do processador)

	Todo o ato de colocar o PROCESSO em EXECUÇÃO e tirá-lo nós chamamos de 
ESCALONAMENTO, este válido para todos os sistemas operacionais.

	Você possui alguns tipos de processos que você vai lidar.

	o CPU BOUND (Orientado ao processador) tempos de execução determinado pelo processador, a exemplo
	  um CÁLCULO, uma compactação de ARquivos, processos que utilizaram muitos recursos do processador.
	  em detrimento dos dispositivos de entrada e saída.
	o I/O BOUND (Processos orientados a ENTRADA X SAÍDA)
          Exemplo - Banco de Dados não ha muita execução a nivel de processador
	  porém exigirá escritas no disco.

	 Trocam informações através de sinais, dependendo do Sistema OPeracional
existem vários sinais, no linux são 64 sinais possíveis.
São as formas como os processos se comunicam e como também nos usuários nos comunicamos.
com os processos.

	 HIERARQUIA
	 Um processo cria um segundo, um processo PAI, ele cria um processo filho
esse também criara outro processo, o que se torna uma árvore onde uns estão vinculados a outros.

	 Ao enviar um sinal para finalizar um processo PAI este também irá finalizar todos seus 
processos filhos.

	 ESTADOS DO PROCESSO
	 o EXECUTANDO (Pode esta sendo processado pelo processador)
	 o ESPERANDO (Pode estar em um estado de espera, significa que ele foi executado, e por algum motivo foi posto na fila de espera.
	   fica aguardando a vez dele para ser colocado novamente em processo.
         o PRONTO (Pronto para ser poso em processamento, porém ficam em estado de pronto)
         o SUSPENDO (Foi submeitido posto em execução, porém eta parado, nesse estado não esta na fila de excução.)
           fica aguardando um sinal ou do usuário, ou de um outro processo par ser 
	   posto novamente na fila.

	 o NÃO SUBMETIDO (Não foram submetidos a execução) esta parado.
	   um executável parado.
	 o COMPLETO (Processo foi executado, realizou toda sua tarefa e concluir,
	   ai ele é finalizado.

CRIAR PROCESSOS
	 Momento que Micro e ligado, abrir um programa, um programa que ao executar, 
chama outro programa, uma tarefa agendada.
 

PROCESSO CONCLUSO
	 Ocorre quando muda de estado para COMPLETO, concluir toda a execução dele
normalmente, foi fechado normamente. pode ocorrer um erro que foi gerado, falha de execução (Erro fatal)
exemplo uma divisão por 0, causaria uma excessão e ele seria interrompido. ou por
cancelamento, envia-se uma sinal para ele ser encerrado.

	Os primeitos sistemas operacionais na década de 50 realizaram processamentos
em lote. como assim:

	Existia uma fila de execução, onde a tarefa era submetida e essa tarefa era
executata, era submetido a execução, este ia para a fila e a saída era enviado a exempolo
para um papel.

	Isso perdurou no brasil até o final dos anos 80, era uma forma de evitar
que o computador ficasse ocioso, acelerava o processamento, diferentes prioridades.
	
	Porém não havia ineração com  a máquina, isso começa a mudar quando vem
os primeiros sistemas oparacionais de tempo compartilhado. (SISTEMAS TIME-SHARING).

	MULTICS
	UNIX 

	Mais afinal o que (TIME-SHARING): 
	Programas a serem executados que são os processos serão colocados em execução.
porém eles serão executados durante uma fatia de tempo, a um intervalo de tempo, ex. 10ms, ao concluir esse tempo, ele irá pegar esse processo e o colocara, 
na fila de execução, coloca no final da fila., fica fazendo esse cículo de execução.
todo esse processo chamamos de ESCALONAMENTO.

	Em processadores atuais já são mais sofesticados, a exemplo, processos 
	CPU BOUND, podem ficar mais tempo em execução, ja processo mais orientados 
a ENTRADA/SAÍDA  I/I BOUND ficam menos tempo em processamento.

	Processos com mais prioridades, esses processo furam digamos assim uma posição
na fila de execução.

	Processo com menos prioridades podem ficar no fim da fila por mais tempo.

	Totos sistemas atuais, LINUX, WINDOWS, MAC funcionam em ambientes multiprocessados
e com tempos compartilhados.

	O LINUX é multiusuário, vem da herança do UNIX, ele é um sistema MULTITAREFA.

	SISTEMAS REAL-TIME

	Sitems de tempo compartilhado um processo pode mandar um sinal para outro, com certa

	Em sitemas REAIS possuem tempos pre-definidos, existem prazos, caso a resposta.
não seja entregue dentro daquele tempo determinado, ocorre uma FALHA.
estes sistemas são usados em vários serviços, onde ha necessidade de existir 
esses prazos definidos.

	- Hospital, sistema opercaional que esta operando um equipamento médido.

	- Adotados em missões criticas

	Sistema que opera dentro de um conjunto de um freio abs de um carro, aviões caça.

     acime=Tempo que você acessou o arquivo
    -30=Menos 30 dias 

     APLICANDO AÇÕES NA BUSCA DO FIND
     
     find /etc/ -maxdepth 2 -name "*.conf" -type f -size -50k -atime -2 -exec cp {} /temp/ \;
     -exec = executa o comando cp para copiar o resultado na pasta tmp, obseve que para finalizar
             comando use a barra invertida e o ;

     find /etc/ -name "*~" -delete (Excui arquivos temporários)



ARQUIVOS COMPACTADOS 

FILEROLER - Gerenciador de compatcação para interface gráfica LINUX.
	
        {Permite gerenciar várias formatos de compactação}

No linux ja existem algumas ferramentas de compactação:

TAR   - Programas para agrupar os arquivos. (não compacta, apenas empacota), porém você pode aplicar
       um compactador em cima, os quais podem ser o GZIP e o BZIP2.

GZIP  - É um formato muito comun, rapido para compactar e rapito para descompactar, taxa de compressão boa.

BZIP2 - Mais lento para compactar, porém possui uma taxa um pouco melhor de compactação.

Também permite trabalhar com formatos ZIP, para isso basta apenas instalar o pacote, como:

Comando: 
	apt install zip unzip rar unrar p7zip  (precisa também instalar o UNZIP)


Exemplos:
TAR:

tar cfv arquivo.tar *.conf  
  c => Criar o arquivo.
  f => Forçar
  v => Verbose (Mostra o que esta fazendo)

Compactando arquivo tar.
  gzip arquivo.tar


Descompactando
  gunzip arquivo.tar.gz

GZIP2
  bzip2 arquivo.tar


AGRUPANDO COM TAR E JA COMPACTANDO

  tar -cfvz arquivo2.tar.gz *.conf 

para usar o BZIP2 basta trocar o z pelo J como:

   tar -cfvj arquivo2.tar.bz2 *.conf


DESCOMPACTANDO (X)

   tar -xfvz arquivo2.tar.gz 

DESCOMPACTANDO BZIP2

   tar -xfvj arquivo2.tar.bz2

DESCOMPACTANDO PARA DENTRO DE UM DIRETORIO 

   tar -xfvj arquivo2.tar.bz2 -C "diretorio"/
 
-C -Descompactar dentro diretório.

PBZIP2 - usa todos os nucleos do seu processador no processo de compactação e descompatação.


CRIAÇÃO DE CONTAS (USUÁRIOS)

Comandos:

	useradd (utilizado para adicionar usuários)
        userdel (comando utilizado para excluir usuários)
        usermod (modificar configurações de usuários)

user -D (mostra configuraões padrões para criação do usuários, caso não estabeleça parâmetros)

Exemplos:

	useradd -c "um usuários qualquer" -d /home/usuario -g 100 -m -s /bin/bash -u 1001 alguem 
         -c = Campo comentários, pode colocar o nome completo do usuário
         -d = Diretório pessoal da nova conta 
         -g = Grupo primário da nova conta (100)
         -G = Caso queira que ele faça parte de outro grupo complementar
         -m = Cria a pasta /home do usuário
         -M = Não cria a pasta /HOME do usuário
         -n = Não cria um grupo com mesmo nome do usuário
         -s = Qual interpletador de comando será usado.
         -u = Defini numero do grupo do usuário (caso não estabeleça ele criara o primeiro dispoinivel.
         -U = Cria um grupo com mesmo nome do usuário

Para verificar se o usuário foi criado use o comando:

	 getent passwd (vai listar os usuários)

DEFININDO UMA SENHA PARA O USUÁRIO

	 passwd "nome do usuário"  so permite sendo usuário root 

LOGANDO COM O USUÁRIO NOVO

         su - "nome do usuário"  (SU PERMITE LOGAR COM NOME DE OUTRO USUÁRIO")
         - => Indica que quando logar ira para a pasta do próprio usuário
