# Cronograma de Estudos — Tecnologia, Linux, Infraestrutura e Segurança

## Objetivo
Este cronograma foi estruturado para manter equilíbrio entre:

- aprofundamento técnico;
- curiosidade tecnológica;
- prática real;
- construção de projetos;
- evolução profissional.

A ideia principal é manter um núcleo forte de aprendizado sem abandonar outras áreas de interesse.

---

# Cronograma Semanal

## Segunda-feira — Linux + Infraestrutura

### Estudar

- systemd
- permissões
- gerenciamento de usuários
- serviços Linux
- logs
- SSH
- automação shell

### Prática

- configurar serviços;
- criar scripts simples;
- troubleshooting;
- gerenciamento de servidores.

---

## Terça-feira — Programação

### Estudar

- lógica de programação
- Python ou PHP
- funções
- APIs
- manipulação de arquivos
- automações

### Prática

- scripts úteis;
- ferramentas administrativas;
- pequenos sistemas;
- automações reais.

---

## Quarta-feira — Redes + Docker

### Estudar

- TCP/IP
- DNS
- VLAN
- roteamento
- Docker
- Docker Compose
- containers

### Prática

- criar laboratórios;
- subir containers;
- integrar serviços;
- testar conectividade e redes.

---

## Quinta-feira — Segurança / OSINT / Forense Digital

### Estudar

# Estrutura Diária Recomendada

Mesmo com pouco tempo:

## 1h a 2h por dia

### Divisão sugerida

- 30% teoria
- 70% prática

Seu perfil tende a aprender mais:

- configurando;
- quebrando ambientes;
- investigando problemas;
- criando laboratórios;
- automatizando tarefas.

# Regra Mais Importante

Evite apenas consumir conteúdo.

Tente sempre:

- construir algo;
- automatizar tarefas;
- criar laboratórios;
- resolver problemas reais;
- documentar descobertas.

# Como Utilizar IA nos Estudos

## Use IA para:

- explicar erros;
- revisar código;
- entender arquitetura;
- sugerir melhorias;
- simular problemas;
- ensinar lógica;
- criar desafios.

## Evite:

- copiar código sem entender;
- depender totalmente da IA;
- pular fundamentos.
